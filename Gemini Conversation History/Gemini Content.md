### 1\. 網頁的身分證與設定 (Head)

```html
<!DOCTYPE html>
<html lang="zh-TW">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>我的個人網站</title>
    <link rel="stylesheet" href="style.css">
</head>
```

  * **`<!DOCTYPE html>`**: 這是給瀏覽器看的第一句話，告訴它：「請用最新的 **HTML5** 標準來讀取這個檔案。」如果沒寫，舊版瀏覽器可能會用奇怪的方式顯示。
  * **`<html lang="zh-TW">`**: 這是整個網頁的根部 (Root)。`lang="zh-TW"` 很重要，它告訴瀏覽器（還有 Google）這個網頁主要是用「繁體中文」寫的。
  * **`<head>`**: 這裡是\*\*「幕後控制室」**。這裡面的東西**不會\*\*直接顯示在網頁畫面上，而是給瀏覽器看的設定。
      * **`charset="UTF-8"`**: 確保中文字能正常顯示，不會變成亂碼。
      * **`viewport ...`**: 這是為了**手機版**準備的關鍵設定。它告訴手機瀏覽器：「請依照螢幕實際寬度來顯示，不要把網頁縮得小小的」。
      * **`<link ... href="style.css">`**: 這行最關鍵！它像是一條繩子，把這個 HTML 檔案跟我們等一下要寫的 CSS 樣式表綁在一起。沒有這行，CSS 寫再多都沒用。

-----

### 2\. 導覽列：為什麼用 `<ul>` 和 `<li>`？

```html
<nav>
    <div class="logo">MyPortfolio</div>
    <ul>
        <li><a href="#about">關於我</a></li>
        <li><a href="#portfolio">作品集</a></li>
        ...
    </ul>
</nav>
```

  * **`<nav>`**: 這是 **「語意化標籤 (Semantic Tag)」**。其實用 `<div>` 也可以，但用 `<nav>` 是為了告訴搜尋引擎（SEO）：「這塊區域是用來導航的」。
  * **`<ul>` (Unordered List)** 與 **`<li>` (List Item)**:
      * **為什麼這樣寫？** 你可能會想，為什麼不直接寫三個連結就好？
      * 在網頁開發的標準慣例中，**選單 (Menu) 本質上就是一份「清單」**。使用列表標籤 `<ul>` 可以讓結構更有條理，之後用 CSS 排版（例如把垂直清單變成水平選單）時也會更容易控制。
  * **`<a href="#about">`**: `<a>` 是超連結 (Anchor)。`#about` 代表這是一個\*\*「錨點」\*\*，點擊後會直接跳到同一個頁面中 `id="about"` 的位置，而不是跳轉到其他網站。

### 3\. 作品集區塊：最複雜的結構

這部分是我們為了實現「卡片 + 標籤 + Hover」功能所設計的結構：

```html
<div class="card photo-card">
    <span class="badge photo">📷 攝影</span>
    <img src="..." alt="...">
    <div class="card-info">...</div>
</div>
```

  * **`<div>` (Division)**:
      * 它是網頁中最通用的\*\*「盒子」**或**「容器」\*\*。它本身沒有特殊意義，就是用來把東西包在一起，方便我們移動或上色。
      * **`class="card photo-card"`**: 我們給了這個盒子兩個名字（Class）。
          * `card`: 用來設定所有卡片共通的樣子（例如大小、圓角）。
          * `photo-card`: 用來設定這張卡片特有的樣子（例如之後可能想針對攝影卡片做特殊處理）。
  * **`<span>`**:
      * 跟 `<div>` 很像，但 `<div>` 是**塊級元素 (Block)**（會自己佔滿一行），而 `<span>` 是**行內元素 (Inline)**（就像文字一樣，不會強迫換行）。
      * **為什麼用它？** 因為我們的「標籤 (Badge)」只是一小塊文字背景，不需要佔滿整行，用 `span` 包起來正好可以用 CSS 把它的背景變色。
  * **`<img src="..." alt="...">`**:
      * **`src`**: 圖片來源。
      * **`alt` (Alternative Text)**: 這是**替代文字**。如果圖片連結壞了，或者視障人士使用螢幕閱讀器時，會唸出這段文字。這對無障礙網頁（Accessibility）非常重要。

### 為什麼不全部用 `<div>` 就好？

你可能會發現，其實把這些標籤全部換成 `<div>`，網頁看起來**一模一樣**（在還沒寫 CSS 之前）。那為什麼要這麼麻煩去分這些標籤呢？

1.  **SEO (搜尋引擎優化)：** Google 的機器人讀不懂圖片，但它讀得懂標籤。當它看到 `<nav>`，它知道這是選單；看到 `<footer>`，它知道這是版權資訊。這有助於你的網站被搜尋到。
2.  **無障礙 (Accessibility)：** 視障人士使用螢幕閱讀器上網時，閱讀器可以透過這些標籤快速跳轉（例如：「直接跳到導覽列」或「直接跳到主要內容區」）。這是一種對使用者友善的表現。
3.  **程式碼可讀性：** 當你的程式碼變得很長時，看到 `<section>` 比看到滿滿的 `<div>` 更容易讓人理解結構。

### 第一階段：歸零與基礎設定 (Reset & Base)

瀏覽器（Chrome, Safari）其實都有自己預設的樣式（例如 `<h1>` 預設字很大、`<body>` 預設有邊距）。為了讓設計完全由我們掌控，第一步通常是「清除預設樣式」，我們稱為 **CSS Reset**。

這其中最重要的一個概念是 **「盒模型 (Box Model)」**。

請在 `style.css` 中輸入（或複製）以下程式碼：

```css
/* 1. 全域設定 (Global Reset) */
* {
    margin: 0;
    padding: 0;
    box-sizing: border-box; /* 關鍵！讓邊框和內距包含在尺寸內 */
}

/* 2. 基礎排版 (Typography) */
body {
    font-family: "Helvetica Neue", Arial, sans-serif; /* 設定字體 */
    line-height: 1.6; /* 行高，讓文字好閱讀 */
    color: #333; /* 文字顏色：深灰色 (不建議用純黑 #000) */
    background-color: #f4f4f4; /* 背景色：淺灰白 */
}

/* 3. 連結樣式 */
a {
    text-decoration: none; /* 去除底線 */
    color: #333;
}

/* 4. 簡單的版面限制 */
/* 讓內容不要黏在螢幕邊緣，且在大螢幕上置中 */
section, header, nav {
    max-width: 1200px;
    margin: 0 auto; /* 上下0，左右自動(置中) */
    padding: 20px;
}
```

#### 💡 這裡有個關鍵技術點：`box-sizing: border-box;`

  * **如果不寫：** 當你設定一個盒子寬度 `100px`，然後加了 `10px` 的內距 (padding)，盒子實際寬度會變成 `120px` (100+10+10)。這會讓排版經常「破版」。
  * **寫了之後：** 寬度 `100px` 就是 `100px`，內距會往內擠，不會把盒子撐大。這對專案管理來說，能大幅減少除錯 (Debug) 的時間。

### 第二階段：導覽列與標題 (Navbar & Header)

接下來我們處理頭部。這裡我們會用到 **Flexbox**，這是現代 CSS 排版的神器。

請接著貼上：

```css
/* --- 導覽列 (Flexbox 佈局) --- */
nav {

    display: flex; /* 讓清單變成橫排 */

}

}
```

### 第三階段：核心功能 —— 融合型卡片 (Card Component)

這是我們這個專案的重頭戲！我們要實作那個「滑鼠移上去顯示資訊」的特效。

這裡的核心技術是 **「定位 (Positioning)」**：

  * **父元素 (卡片)** 設為 `relative`（相對定位）：作為基準點。
  * **子元素 (標籤、資訊層)** 設為 `absolute`（絕對定位）：依照父元素來決定位置。

請接著貼上：

```css
/* --- 作品集網格 (CSS Grid) --- */
.portfolio-grid {
    display: grid;
    /* 自動排版魔法：每個格子最小 300px，剩下空間自動平分 */
    grid-template-columns: repeat(auto-fit, minmax(300px, 1fr));
    gap: 20px; /* 格子之間的空隙 */
}

/* --- 卡片本體 --- */
.card {
    position: relative; /* 關鍵：讓裡面的絕對定位元素知道以誰為基準 */
    height: 300px; /* 固定高度 */
    border-radius: 10px; /* 圓角 */
    overflow: hidden; /* 超出圓角的圖片切掉 */
    box-shadow: 0 4px 8px rgba(0,0,0,0.1);
    cursor: pointer; /* 滑鼠變成手指 */
}

/* --- 卡片內的圖片 --- */
.card img {
    width: 100%;
    height: 100%;
    object-fit: cover; /* 保持圖片比例填滿，不變形 */
    transition: transform 0.3s ease; /* 圖片放大的動畫設定 */
}

/* --- 左上角的標籤 (Badge) --- */
.badge {
    position: absolute; /* 絕對定位 */
    top: 10px;
    left: 10px;
    padding: 5px 10px;
    border-radius: 20px;
    color: white;
    font-size: 0.8rem;
    z-index: 10; /* 確保它浮在最上面 */
}

/* 針對不同類型給不同顏色 */
.badge.photo { background-color: rgba(0, 0, 0, 0.7); }
.badge.academic { background-color: rgba(30, 60, 120, 0.8); }

/* --- 隱藏的資訊層 (Overlay) --- */
.card-info {
    position: absolute;
    bottom: 0;
    left: 0;
    width: 100%;
    padding: 20px;
    background: linear-gradient(to top, rgba(0,0,0,0.9), transparent); /* 漸層黑底 */
    color: white;
    
    /* 初始狀態：隱藏 */
    opacity: 0; 
    transform: translateY(20px); /* 稍微往下沉一點 */
    transition: all 0.3s ease; /* 過渡動畫 */
}

/* --- 互動特效 (Hover Effects) --- */

/* 當滑鼠移到卡片(.card)上時，改變圖片(.img)的樣子 */
.card:hover img {
    transform: scale(1.1); /* 圖片微微放大 */
}

/* 當滑鼠移到卡片(.card)上時，改變資訊層(.card-info)的樣子 */
.card:hover .card-info {
    opacity: 1; /* 顯示 */
    transform: translateY(0); /* 浮上來 */
}
```

```css
選取器 (Selector) {
```

  * **屬性**：你要改變它的「什麼特徵」？（例如：顏色、寬度、字體）。

### 第一區塊：全域設定 (Global Reset)

這段程式碼被稱為「CSS Reset」或「正規化」。

```css
* {
    margin: 0;
    padding: 0;
    box-sizing: border-box;
}
```

#### 1\. `*` (星號選取器)

  * **意義**：這是「萬用選取器 (Universal Selector)」。
  * **用法**：它代表網頁上**所有的**元素（包含 `html`, `body`, `div`, `h1`, `p`...全部）。
  * **為什麼這樣寫**：我們希望接下來設定的規則，預設套用到每一個東西上面，省得一個一個寫。

#### 2\. `margin: 0;` 與 `padding: 0;`

要理解這兩行，必須先理解 CSS 最重要的概念：**盒模型 (Box Model)**。網頁上每一個元素其實都是一個長方形的盒子。

  * **Margin (外距)**：盒子**外面**跟別人保持的距離（像是人與人的社交距離）。
  * **Padding (內距)**：盒子**裡面**內容與邊框的距離（像是畫框與畫布之間的留白）。
  * **為什麼設為 0？**
      * 瀏覽器（Chrome, Safari）預設會給某些標籤加距離。例如 `<body>` 預設有 8px 的邊距，`<h1>` 標題預設有很大的上下留白。
      * 如果不歸零，你的網頁邊緣會莫名其妙出現白邊，或者排版無法對齊。這動作叫做「歸零」，讓我們可以在一張乾淨的畫布上重新設計。

#### 3\. `box-sizing: border-box;` (最關鍵的一行)

  * **意義**：改變盒子尺寸的計算方式。
  * **預設值 (content-box)**：假設你設寬度 100px + 內距 20px + 邊框 5px。實際寬度會變成 `100 + 20 + 20 + 5 + 5 = 150px`。這會讓新手非常頭痛，因為版面很容易被撐爆。
  * **設定值 (border-box)**：你設寬度 100px，它就是 100px。內距和邊框會**往內擠**，不會把盒子撐大。
  * **為什麼這樣寫**：為了讓專案管理直觀化。身為開發者，當我說「寬度 300px」，我希望它就是 300px，不要因為我加了邊框還要自己算數學扣回去。

-----

### 第二區塊：基礎排版 (Base Styles)

這裡我們針對 `body` 進行設定。因為 HTML 具有**繼承 (Inheritance)** 的特性，設定在 `body` 上的字體和顏色，會自動傳承給裡面的 `p`, `div`, `span` 等子孫元素，除非子孫自己有另外設定。

```css
body {
    font-family: "Helvetica Neue", Arial, sans-serif;
    line-height: 1.6;
    color: #333;
    background-color: #f4f4f4;
}
```

#### 1\. `font-family: ...` (字體家族)

  * **語法**：`"首選字體", 次選字體, 通用類別;`
  * **運作邏輯**：瀏覽器會依序尋找。
    1.  先看電腦有沒有 `"Helvetica Neue"`（Mac 常見字體）。
    2.  沒有的話，找 `Arial`（Windows 常見字體）。
    3.  再沒有，就用電腦預設的 `sans-serif`（無襯線體，就是筆畫沒有裝飾、乾淨的字體）。
  * **為什麼這樣寫**：確保在不同電腦上，文字都能保持乾淨、現代的風格（無襯線體比較適合螢幕閱讀）。

#### 2\. `line-height: 1.6;` (行高)

  * **意義**：每一行文字的高度。
  * **數值**：`1.6` 代表字體大小的 1.6 倍。如果字體是 16px，行高就是 25.6px。
  * **為什麼這樣寫**：預設的行高通常太擠（約 1.2）。設定 1.5 \~ 1.6 是最適合人類閱讀長篇文章的舒適度，這能大幅提升網站的質感。

#### 3\. `color: #333;` (文字顏色)

  * **語法**：使用 **十六進位碼 (Hex Code)**。
      * `#000000` 是純黑，`#FFFFFF` 是純白。
      * `#333` 是 `#333333` 的縮寫，代表深灰色。
  * **為什麼這樣寫**：**設計圈的潛規則**——盡量不要用「純黑 (\#000)」。純黑對比太強，在發光螢幕上閱讀久了眼睛會累。深灰色 (`#333`) 看起來既像黑色，又比較柔和。

#### 4\. `background-color: #f4f4f4;` (背景色)

  * **意義**：設定整個網頁的底色。
  * **數值**：`#f4f4f4` 是一種非常淡的灰白色。
  * **為什麼這樣寫**：純白背景有時候會太刺眼。淡灰色可以讓放在上面的白色卡片（我們待會要做）看起來更有層次感（Card Design 的基礎）。

### 第三區塊：連結樣式 (Links)

```css
a {
    text-decoration: none;
    color: #333;
}
```

#### 1\. `text-decoration: none;`

  * **意義**：文字裝飾。
  * **為什麼這樣寫**：瀏覽器預設的超連結會有「藍色底線」。這在現代網頁設計中通常被認為比較老氣。我們用 `none` 把底線拿掉，之後可以再用其他方式（如 Hover 變色）來提示它是連結。

#### 2\. `color: #333;`

  * **為什麼這樣寫**：預設的連結是亮藍色，點過後變紫色。這會破壞我們網站的配色。我們強制把它變成跟普通文字一樣的深灰，保持視覺統一。

-----

### 第四區塊：版面限制與置中 (Layout Utility)

這是一個「群組選取器」，一次選取三個標籤。

```css
section, header, nav {
    max-width: 1200px;
    margin: 0 auto;
    padding: 20px;
}
```

#### 1\. `section, header, nav`

  * **用法**：用逗號 `,` 分隔，代表「這三個傢伙都適用以下規則」。

#### 2\. `max-width: 1200px;` (最大寬度)

  * **意義**：這個容器最寬只能長到 1200px。
  * **為什麼這樣寫**：現在的螢幕都很大（例如 1920px 或 4K）。如果文字從最左邊一路寫到最右邊，使用者的脖子會扭得很酸。限制最大寬度，讓內容集中在中間，是提升閱讀體驗的關鍵。

#### 3\. `margin: 0 auto;` (水平置中神器)

  * **語法**：`margin: 上下值 左右值;`
      * `0`：上下外距為 0。
      * `auto`：左右外距「自動」。
  * **運作邏輯**：當左邊也是 auto，右邊也是 auto，瀏覽器就會把剩餘的空間平分給左右兩邊，結果就是**盒子會正正地擺在畫面中間**。
  * **必備條件**：這個元素必須要有寬度（或是設定了 max-width），`auto` 才算得出剩餘空間有多少。

#### 4\. `padding: 20px;`

  * **為什麼這樣寫**：這是為了手機版。如果使用者的螢幕小於 1200px，內容會佔滿寬度。這時候左右如果沒有留一點 `padding`，文字就會黏在手機螢幕邊緣，很難看。這留了 20px 的「呼吸空間」。

### 筆記重點總結 (Takeaway)

1.  **`* { box-sizing: border-box; }`**：這是現代 CSS 的起手式，讓寬度計算符合人類直覺。
2.  **`body` 的設定**：決定了整個網站的基調（字體、顏色、行高）。
3.  **顏色選擇**：避免純黑，使用深灰 (`#333`) 更護眼；背景用淡灰 (`#f4f4f4`) 來襯托白卡片。
4.  **`margin: 0 auto;`**：這是讓區塊在螢幕上「水平置中」最常用的技巧。
